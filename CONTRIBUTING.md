# Contributing

## Reporting Issues

Open an issue for:
- Documentation errors or unclear instructions
- Suggestions for additional guardrail patterns
- Compatibility issues with specific tools or CI systems

## Pull Requests

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Run `/socket` on any dependency changes (practice what we preach)
5. Submit a PR with a clear description

## Documentation Style

- Keep examples concrete and copy/paste ready
- Avoid marketing language
- Test all code snippets before submitting
- Ensure Markdown renders correctly on GitHub

## Code of Conduct

Be respectful and constructive. Focus on helping developers ship secure code.
